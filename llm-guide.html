100xFenok LLM Site Guide

This document is a lean, machine-readable guide for LLM crawlers and developer agents. The canonical JSON block is embedded between LLM_GUIDE_START and LLM_GUIDE_END.

Human Developer Summary

  1. Cloudflare Workers Environment: Root path is served via Workers at https://100xfenok.etloveaui.workers.dev/.
  2. Next.js App Router Architecture: Replaces the old index/loadPage SPA framework. Routing is governed dynamically by routes.ts.
  3. Legacy Bridge (DEC-256): Any legacy vanilla HTML asset resides inside public/ and is served behind standard Next.js page shells using iframe RouteEmbedFrame. Direct root requests are gated or redirected.
  4. Public Data Namespaces: LLMs can crawl structured pricing/analytical data directly from flat JSON files registered under the /data/ namespace.

Ambiguity Flags

  • Flag 1: routes.ts Dynamic Slugs: routes.ts contains dynamic parameter functions like stock(ticker) and etf(ticker). Crawler maps use /stock/{ticker} and /etfs/{ticker} wildcard patterns.
  • Flag 2: workbench Route Status: ROUTES.workbench resolves to /workbench, but the route remains pending final owner decision. It is listed in core_routes with an explicit status note.
